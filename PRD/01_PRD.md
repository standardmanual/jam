# JAM! 유저 모바일 웹 — PRD (Product Requirements Document)

> 생성일: 2026-07-09
> 생성 도구: Show Me The PRD

---

## 1. 제품 개요

### 한 줄 요약
운동 후 Strava 연동 한 번으로 디지털 배지를 수확하고, 실물 패치를 구매할 수 있는 피지털 게이미피케이션 모바일 웹 앱.

### 해결하는 문제
운동 기록 앱(Strava, Garmin)은 이미 충분하지만, **운동에 대한 의미 있는 보상**이 없다. JAM!은 기존 트래킹 습관을 바꾸지 않고 저녁에 앱을 열면 배지와 아이템이 쌓이는 "보상 수확" 경험을 제공한다.

### 핵심 가치
- **Tracker-less**: 자체 GPS 트래킹 없음. 기존 Strava 데이터 그대로 활용
- **10초 첫 보상**: 가입 → Strava 연동 → 첫 배지 수령까지 마찰 제로
- **피지털 커머스**: 배지 달성 시에만 구매 가능한 실물 자수 패치 (돈이 있어도 아무나 살 수 없음)

## 2. 사용자

### 주요 사용자 (Phase 1 기준)
- **누구**: 이미 Strava를 사용하는 자전거/달리기/등산/걷기 활동자
- **상황**: 운동 후 귀가해서 저녁에 스마트폰으로 앱 확인
- **목표**: 오늘 운동으로 어떤 배지가 생겼는지 확인하고 실물 패치 구매 자격 쌓기

### 페르소나
1. **자전거 자출족 (2030)**: 출퇴근 라이딩. 공유자전거 or 개인 자전거. 누적 거리 배지 목표.
2. **등산 인증샷러 (3040)**: 주말 산행. 봉우리 인증 배지 + 가방 꾸미기 아이템 패치.
3. **러닝 크루 (2030)**: 소셜 중심. 인스타 공유 카드 생성으로 SNS 확산.
4. **일상 걷기러 (4050)**: 최대 DAU 기반. 일별/주별 걸음 달성 배지.

### 핵심 사용자 시나리오

**시나리오 1 — 첫 방문 (10초 보상 플로우)**
1. 링크로 JAM! 접속 → 구글 로그인
2. 활동 종목(자전거/달리기/등산/걷기), 지역 선택
3. Strava 연동 버튼 탭 → 연동 완료
4. 이후 Strava 활동이 동기화될 때마다 조건 달성 시 배지 자동 발급
5. 첫 동기화 후 "배지 획득!" 알림 → 배지 컬렉션 화면으로 이동

**시나리오 2 — 일상 사용 (매일 저녁)**
1. 앱 열기 → 홈 피드에서 "오늘 활동으로 배지 1개 업데이트" 알림 확인
2. 나의 배지 탭 → 새 배지 상세 보기
3. 인스타 공유 카드 생성 → 공유

**시나리오 3 — 실물 패치 구매**
1. 배지 상세 화면에서 "실물 패치 구매 가능" 뱃지 확인
2. D2C 스토어로 연결 → 해당 패치 장바구니 활성화

## 3. 핵심 기능

| 기능 | 설명 | Phase | 복잡도 |
|------|------|-------|--------|
| 구글 소셜 로그인 | 비밀번호 없이 구글 계정으로 가입/로그인 | 1 | 간단 |
| 프로필 설정 | 활동 종목(복수), 지역 선택 | 1 | 간단 |
| Strava OAuth 연동 | Strava 계정 연결. 로그인 시점 + 정기(일 1회) 동기화 두 가지 방식 | 1 | 복잡 |
| 홈/피드 | 오늘의 활동 요약, 최신 배지 업데이트, 커뮤니티 피드 | 1 | 간단 |
| 나의 배지 컬렉션 | 액티비티/아이템 배지 목록, 아이템북 진척도 | 1 | 간단 |
| 배지 상세 + 인스타 공유 카드 | 배지 정보, 3D 포토카드 생성, 공유, POI 위치 외부 지도앱 아웃링크 | 1 | 보통 |
| POI 사후 GPS 매칭 | Strava 동기화 후 자동 POI 인증 체크 (반경 50m). 지도 화면 없음 | 2 | 복잡 |
| 인벤토리 | 아이템 배지 50슬롯 관리 | 3 | 보통 |
| 플리마켓 (메뉴만) | 거래 UI 진입점만 노출. 실제 거래 기능은 Phase 4 | 3 | 간단 |

## 4. 화면 구성 (Mobile Web)

```
[홈/피드]
  ├── 오늘의 활동 요약 카드 (Strava 동기화 기준)
  ├── 새 배지 획득 알림
  └── 최신 커뮤니티 피드 (Phase 2+)

[나의 배지]
  ├── 액티비티 배지 그리드
  ├── 아이템 배지 그리드
  ├── 아이템북 목록 (진척도 표시)
  └── 배지 상세 → 인스타 공유 카드 생성
      (POI 위치 "지도에서 보기" 아웃링크는 Phase 2 — POI 데이터 추가 후 활성화)

[인벤토리] (Phase 3)
  ├── 50슬롯 아이템 인벤토리
  └── [플리마켓] 메뉴 진입점 (기능 잠금 상태)

[프로필]
  ├── 활동 종목 / 지역 설정
  ├── Strava 연동 상태
  └── 획득 배지 통계
```

## 5. 성공 기준 (Phase 1)

- [ ] Strava 연동 완료 후 첫 동기화 시 배지 자동 발급 정상 동작
- [ ] 로그인 시점 동기화: 로그인 후 30초 이내 최신 Strava 활동 반영
- [ ] 정기 동기화: 일 1회 자동 실행, 새 활동 감지 시 배지 발급
- [ ] 인스타 공유 카드 이미지 생성 시간 3초 이내
- [ ] 모바일 웹 (iOS Safari, Android Chrome) 정상 동작
- [ ] 실제 URL로 외부인이 접속하여 회원가입부터 배지 수령까지 완료 가능

## 6. 안 만드는 것 (Out of Scope — Phase 1)

- **자체 GPS 트래킹 UI** — 이유: JAM! 핵심 철학 "Tracker-less". Strava/Garmin이 이미 잘 함
- **과거 데이터 소급 분석** — 이유: 연동 시점 이후 활동만 인정. 소급 적용 없음
- **인앱 지도 화면** — 이유: POI 위치는 배지 상세에서 외부 지도앱(카카오맵/구글맵) 아웃링크로 대체
- **Garmin 연동** — 이유: Strava PoC 먼저 검증 후 추가. 동시 개발 시 복잡도 2배
- **Apple Health 연동** — 이유: 웹 앱에서 직접 접근 불가. 별도 네이티브 앱 필요
- **실제 플리마켓 거래** — 이유: DAU 기반 잠금 해제 조건 미충족 상태에서 먼저 열면 의미 없음
- **드랍/픽업 시스템** — 이유: 서울 DAU 3만명 달성 후 전국 오픈 예정
- **P2P 실시간 채팅** — 이유: 거래 기능 자체가 Phase 4
- **인앱 결제** — 이유: D2C는 외부 쇼핑몰(별도 URL)로 연결
