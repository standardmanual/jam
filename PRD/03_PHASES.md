## Phase 1: 온보딩 + 배지 코어 (예상 4-5주)

### 목표
Strava를 연동한 이후 새로운 활동이 동기화될 때마다 배지가 자동 발급되고, 인스타에 공유할 수 있는 포토카드를 생성할 수 있는 상태.

"운동하면 JAM! 앱에서 배지가 쌓인다"는 핵심 가치를 실제로 검증.

### 기능
- [ ] **구글 소셜 로그인**: Supabase Auth + Google OAuth 연동
- [ ] **프로필 설정**: 활동 종목(자전거/달리기/등산/걷기 복수 선택), 지역(시/도)
- [ ] **Strava OAuth 연동**: Strava 계정 연결 버튼 → OAuth 인증 → access_token 저장
- [ ] **로그인 시점 동기화**: 로그인 완료 시 Strava 최신 활동 즉시 가져오기 → 배지 조건 매칭
- [ ] **정기 동기화**: 일 1회 백그라운드 실행 → 새 Strava 활동 감지 → 배지 자동 발급
- [ ] **배지 자동 발급**: 조건 충족 배지 즉시 생성 → UserActivityBadge 저장
- [ ] **홈 피드**: 오늘 활동 요약 카드, 최근 획득 배지 목록
- [ ] **나의 배지 컬렉션**: 액티비티 배지 그리드, 아이템북 진척도
- [ ] **배지 상세 화면**: 배지 정보, 획득 조건, 희귀도
- [ ] **인스타 공유 카드 생성**: 3D 배지 이미지 + 고유 번호 합성 이미지 → 다운로드/공유
- [ ] **프로필 화면**: Strava 연동 상태, 활동 통계, 마지막 동기화 시각 표시

### 사용하는 데이터 엔티티
- User, StravaConnection, Badge, UserActivityBadge, ItemBook (읽기 전용)

### 인증
- 구글 소셜 로그인 (Supabase Auth)

### "진짜 제품" 체크리스트
- [ ] 실제 Supabase DB 연결 (목업 데이터 X)
- [ ] 실제 구글 OAuth 로그인 (하드코딩된 테스트 계정 X)
- [ ] 실제 Strava OAuth 연동 (테스트용 mock API X)
- [ ] Vercel에 배포된 실제 URL로 접속 가능
- [ ] 외부인이 URL로 들어와 회원가입 → Strava 연동 → 배지 수령까지 완료 가능
- [ ] iOS Safari, Android Chrome 모두 정상 동작

## Phase 2: POI 사후 인증 (예상 3-4주)

### 전제 조건
- Phase 1이 Vercel에 안정적으로 배포되어 있고, 실제 Strava 동기화가 정상 동작 중

### 목표
Strava 활동이 동기화될 때 자동으로 POI 통과 여부를 검증하고 인증 배지를 발급하는 상태.
별도 지도 화면 없이, 배지 상세 화면에서 "이 POI 위치 보기" → 카카오맵/구글맵 앱으로 아웃링크.

### 기능
- [ ] **사후 GPS ↔ POI 매칭**: Strava 동기화 시 활동 경로와 POI 반경 50m 교차 검증
- [ ] **POI 인증 배지 자동 발급**: 통과 확인 시 해당 POI 연결 배지 발급
- [ ] **배지 상세 → 외부 지도 아웃링크**: POI 좌표 기반으로 카카오맵(`kakaomap://`) / 구글맵(`https://maps.google.com/?q=`) 딥링크 생성
- [ ] **Garmin 연동 추가** (옵션): Phase 1 Strava 연동과 동일한 패턴으로 추가

### 추가 데이터
- POI 테이블 (어드민 도구로 사전 등록)
- UserActivityBadge에 `triggered_by_poi_id` 필드 추가

### 주의사항
- Strava API에서 GPX 경로 데이터 제공 여부 확인 필요 (기본 활동 API vs 상세 streams API 차이)
- 카카오맵 앱 미설치 시 구글맵 웹으로 폴백하는 아웃링크 처리 필요

### 통합 테스트
- Phase 1 (Strava 연동, 배지 발급, 홈 피드)이 여전히 정상 동작하는지 확인
- POI 매칭이 Phase 1 배지 발급 로직과 충돌하지 않는지 확인

## Phase 3: 인벤토리 + 플리마켓 UI (예상 3-4주)

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중

### 목표
아이템 배지가 드랍되어 인벤토리에 쌓이고, 플리마켓 진입 화면까지 보이는 상태.
(실제 거래 기능은 Phase 4에서 구현)

### 기능
- [ ] **아이템 배지 드랍 시스템**: 활동 완료 후 확률 드랍 로직 (희귀도별 확률 테이블)
- [ ] **인벤토리 화면**: 50슬롯 시각화, 아이템 배지 그리드, 만료 임박 알림
- [ ] **인벤토리 아이템 상세**: 배지 정보, 시리얼 넘버, 만료일
- [ ] **플리마켓 메뉴 진입**: 화면은 있지만 "준비 중" 상태로 노출. 거래 기능 비활성.
- [ ] **아이템북 완성 체크**: 필요한 아이템 배지 모두 획득 시 아이템북 완성 알림

### 추가 데이터
- Inventory, InventoryItem 활성화

### 주의사항
- 아이템 배지 드랍 확률 테이블은 어드민 도구로 조정 가능해야 함
- 만료 30일 규칙: expires_at 이후 자동 소멸 → 잼 포인트 환전 (포인트 시스템은 Phase 4)

## Phase 4+: 고도화 (미정)

### 목표
플리마켓 실제 거래 + P2P 교환 + 잼 포인트 시스템 + SNS 기생 성장

### 기능 (검토 중)
- [ ] 1:1 아이템 배지 교환 (TradeRequest 활성화)
- [ ] 잼 포인트 시스템 (아이템 만료 시 환전, 특별 드랍 사용)
- [ ] Strava 활동 사진첩에 JAM! 배지 이미지 자동 삽입
- [ ] 카카오톡 딥링크 거래 알림
- [ ] 드랍/픽업 시스템 (서울 DAU 3만명 달성 후 전국 오픈)

### 주의사항
- Phase 3까지의 DAU/잔류율 데이터를 보고 우선순위 재결정
- 거래 봇/작업장 방어 정책 필요 (Trade 테이블에 rate_limit, flagged 필드 추가 예정)
