## Phase 8: 아이템북 완성 루프 (예상 2주)

### 목표
아이템북 레시피를 모두 채우면 보상 배지가 자동 발급되는 완성 루프 구현.
현재 아이템북 UI는 있지만 완성 조건 충족 시 아무 일도 일어나지 않는 상태.

### 기능
- **아이템북 완성 감지**: 인벤토리에 required_item_badge_ids가 모두 갖춰지면 자동 감지
- **완성 시 보상 발급**: reward_badge가 있으면 user_activity_badges에 자동 INSERT
- **완성 알림**: "아이템북 완성! 보상 배지를 획득했어요" 화면 내 알림
- **완성된 아이템북 UI**: 아이템북 목록에서 완성 여부 표시 (뱃지/그레이스케일)
- **Strava 동기화 시 트리거**: 배지 발급 직후 아이템북 완성 체크 파이프라인 추가

### 비즈니스 가치
아이템북은 JAM!의 핵심 콘텐츠 소비 루프. 완성 → 보상 → 새 목표 생성의 사이클이 없으면 인벤토리가 쌓여도 의미 없음.

### 전제 조건
Phase 3 (인벤토리), Phase 6 (아이템 드랍)

## Phase 9: 아이템 만료 + JAM 포인트 (예상 3주)

### 목표
인벤토리 아이템의 30일 만료 정책 실행 + 만료 시 JAM 포인트로 자동 환전.

### 기능
- **만료 Cron Job**: 매일 자정 `expires_at < NOW()` 아이템 자동 처리
- **JAM 포인트 테이블**: `user_points` (user_id, balance, updated_at)
- **포인트 내역 테이블**: `point_transactions` (user_id, amount, reason, created_at)
- **만료 환전**: 아이템 rarity별 포인트 환산 (common: 10P, rare: 30P, legendary: 100P, mythic: 500P)
- **포인트 잔액 표시**: 프로필 화면에 JAM 포인트 잔액 표시
- **포인트 사용처** (이후 Phase 연계):
  - 특별 드랍 이벤트 참가권
  - 인벤토리 슬롯 확장 (기본 50 → 최대 100)

### 비즈니스 가치
아이템 만료가 "손해"가 아닌 "환전"이 되면 유저 이탈 방지. 포인트 시스템은 이후 인앱 경제의 기반.

### 전제 조건
Phase 3 (인벤토리 + 아이템 만료 필드)

## Phase 10: 푸시 알림 / 인게이지먼트 (예상 3주)

### 목표
핵심 이벤트 발생 시 유저에게 알림을 보내 재방문율 향상.

### 기능
- **Web Push 구독**: Service Worker + Push API (PWA 필수 선행)
- **알림 종류**:
  - 배지 발급: "오늘 라이딩으로 새 배지를 획득했어요!"
  - 드랍 픽업됨: "당신이 드랍한 [배지명]을 누군가 픽업했어요"
  - 아이템북 완성: "[아이템북 이름] 완성! 보상 배지를 확인하세요"
  - 만료 임박: "아이템 3개가 7일 후 만료됩니다"
  - Strava 동기화 완료: "오늘의 활동이 동기화됐어요. 새 배지 1개!"
- **알림 설정**: 유저별 알림 ON/OFF 선택 (프로필 화면)
- **어드민**: 전체 공지 푸시 발송 기능

### 비즈니스 가치
JAM!은 "저녁에 앱을 열면 배지가 쌓이는" 경험이 핵심. 알림 없이는 재방문 동기가 약함. DAU 직결 기능.

### 전제 조건
PWA 설정 (manifest.json + Service Worker), HTTPS 필수

## Phase 11: 커뮤니티 피드 + 소셜 공유 (예상 4주)

### 목표
다른 유저의 배지 획득, 드랍/픽업 활동이 피드에 노출되어 경쟁/참여 동기 유발.

### 기능
- **팔로우 시스템**: `user_follows` 테이블 (follower_id, following_id)
- **활동 피드**: 팔로우한 유저의 배지 획득, 드랍 완료, 아이템북 완성 노출
- **전체 피드**: 팔로우 없어도 볼 수 있는 지역 기반 전체 피드
- **인스타 공유 카드 고도화**:
  - 배지 + 일련번호 + "JAM! 앱에서 확인" QR코드 합성
  - 공유 후 링크로 해당 배지 상세 직접 진입
- **랭킹**: 이번 달 배지 획득 수 기준 지역별 랭킹 (Top 10)
- **유저 프로필 공개**: `/u/[userId]` — 공개 배지 컬렉션 페이지

### 비즈니스 가치
SNS 확산이 JAM! 신규 유입의 핵심 채널. 공유 카드 → 링크 → 가입의 바이럴 루프.

### 전제 조건
Phase 1 (배지 발급), Phase 10 (알림 — 팔로우 알림 연계)

## Phase 12: 실물 패치 D2C 연동 (예상 2주)

### 목표
배지 달성 시에만 구매 가능한 실물 자수 패치 구매 플로우 연결.
"돈이 있어도 아무나 살 수 없음"이 JAM! 핵심 차별점.

### 기능
- **패치 구매 가능 배지 표시**: 배지 상세에 "실물 패치 구매 가능" 뱃지 (patch_available = true)
- **외부 D2C 스토어 연결**: 배지별 패치 상품 URL로 아웃링크
  - 배지 달성 여부 검증 후 버튼 활성화 (미달성 시 잠금 + 조건 안내)
- **구매 인증 코드**: 달성 유저에게 1회용 할인/인증 코드 발급 (어드민 생성)
- **패치 컬렉션 화면**: 구매 가능/구매 완료 패치 모아보기
- **어드민**: 패치 상품 URL 관리, 인증 코드 발급

### 비즈니스 가치
JAM!의 첫 번째 수익 모델. DAU를 매출로 전환하는 핵심 퍼널.

### 전제 조건
Phase 1 (배지 발급), D2C 쇼핑몰 외부 구축 필요

## Phase 13: P2P 플리마켓 (예상 5주)

### 목표
유저 간 아이템 배지 1:1 교환 거래. 기존 Phase 3에서 "준비 중"으로 잠근 플리마켓 실제 오픈.

### 기능
- **거래 요청**: `trades` 테이블 활성화 (이미 스키마 존재)
  - 내 아이템 제시 + 상대 아이템 요청 → 상대에게 거래 요청 전송
- **거래 수락/거절**: 요청받은 유저가 수락/거절
- **원자 교환**: 트랜잭션으로 두 아이템 동시 교환 (중간 실패 없음)
- **거래 알림**: "거래 요청이 왔어요" / "거래가 수락됐어요" 알림
- **거래 이력**: 내가 보낸/받은 거래 내역 조회
- **사기 방지**:
  - 거래 확인 후 24시간 내 취소 불가
  - 동일 아이템 중복 요청 차단
  - rate_limit: 1일 최대 10건

### 비즈니스 가치
커뮤니티 활성화 + 희귀 아이템 수요/공급 생성 → 플리마켓 자체가 콘텐츠.

### 전제 조건
Phase 3 (인벤토리), Phase 10 (알림), Phase 11 (유저 간 관계)

## Phase 14: 지도 고도화 + POI 확장 (예상 3주)

### 목표
드랍/픽업 지도 UX 개선 + T3 POI (체육관, 공원 입구 등) 확장.

### 기능
- **마커 클러스터링**: 줌아웃 시 밀집 마커 그룹화
- **드랍 히트맵**: 내 위치 기준 드랍 밀도 시각화
- **드랍 이력**: 내가 드랍한 아이템 현황 (픽업됐는지 확인) + 내가 픽업한 이력
- **T3 POI 추가**: 헬스장, 공원 입구, 학교 등 생활 POI
  - OSM 태그: `leisure=fitness_centre`, `leisure=park`, `amenity=school`
- **POI 거리 표시**: 지도 하단에 "가장 가까운 POI: CU 서초점 32m" 상시 표시
- **오프라인 대응**: 위치 없을 때 주변 드랍 목록 텍스트 뷰로 폴백

### 전제 조건
Phase 7 (드랍/픽업 + OSM T2)
