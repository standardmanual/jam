## 1. 배경 및 목표

### 현행 시스템 분석 결과 (2026-07-21 전체 소스코드 분석)

**이미 갖춰진 것 (재사용):**
- `factions` 테이블 = 세계관 10개 시드 완료 (019) — 세계관 문서와 동일한 이름, `drop_weight`·`is_active`·`sort_order` 보유
- `item_books` 100권 + `faction_id` 연결, 아이템배지 ~900종 시드 완료 (019, 027로 고아 배지 정리, 028로 활성화)
- `user_item_book_slots` / `user_item_book_completions` — 슬롯 장착·완성 추적
- `abusing_policy` 싱글톤 테이블 + 어드민 편집 화면 — **설정값 관리 패턴의 선례**
- 어드민: factions/itembooks/badges CRUD, 시뮬레이터
- 서비스 UI: 인벤토리(일련번호 표시), 아이템북 상세(슬롯 그리드), 조합

**문제 (업그레이드 대상):**
- 드랍엔진 v1이 세계관 구조를 **전혀 활용하지 않음** — 활동당 80% 확률, rarity 추첨 후 전체 활성 북의 플랫 풀에서 완전 랜덤
- 결과: 서사 단절(직전 드랍과 무관한 세계관), 분산(100권을 동시에 0~9% 진행), 완성 페이싱 통제 불가
- `inventory_items.serial_number`가 SERIAL(순차) — 발견 순서 서열 노출

### Phase 11 목표

`BADGE_ENGINE_UNIFIED.md` §3의 3레이어 드랍엔진 v2를 실제 서비스에 적용한다:

1. **Layer 1**: 활동당 최소 1개 확정 드랍 + 변동 희귀도(60/28/9/3) + rare+ pity + 보너스 드랍 + 복귀 보너스
2. **Layer 2**: 세계관 모멘텀(50/25/15) — factions 기반, 하드캡·선택 UI 없이 가중치로만 집중
3. **Layer 3**: 완성 페이싱 — 완성도 감쇠, 완성 북 잔류(×0.3), 마지막 조각 pity
4. **맥락 오버라이드**: 온도·시간대·복귀·고도 기반 (강수는 이번 범위 제외 — 외부 날씨 API 추후)
5. **일련번호 무작위화**: 1~999,999 난수
6. **어드민**: 드랍 파라미터 설정 테이블(`drop_policy` 싱글톤) + 편집 화면 + factions 인접 관리

## 2. 확정된 결정사항

| 항목 | 결정 |
|------|------|
| 파라미터 관리 | **어드민 설정 테이블** (`drop_policy` 싱글톤, abusing_policy 패턴) — 배포 없이 튜닝 |
| 배포 방식 | **단계 릴리즈** — 스키마 → Layer1 → Layer2·3 → 맥락 순 |
| 기존 유저 모멘텀 | **초기화 불필요** — 현재 유저는 전원 테스트 유저. `user_drop_state` 빈 상태 시작 |
| 강수 오버라이드 | **이번 범위 제외** — 온도(Strava average_temp)·시간대·복귀·고도만. 날씨 API는 추후 |
| 세계관 저장 | 신규 `worlds` 테이블 만들지 않음 — **기존 `factions` 재사용** (인접 관계만 추가) |

## 3. 사용자 경험 변화

| 시점 | v1 (현재) | v2 (Phase 11 후) |
|------|-----------|------------------|
| 활동 완료 | 80% 확률로 랜덤 아이템 (20% 꽝) | **무조건 1개+** (15% 확률 2개), 희귀도가 서스펜스 |
| 연속 드랍 | 매번 무관한 세계관 | ~75%가 직전 세계관·인접 세계관 — "이야기가 이어짐" |
| 새벽 러닝 | 무관한 아이템 | 60% 확률로 비트 마에스트로/셔터 마피아 파편 — "성취의 증거" |
| 7일 쉬고 복귀 | 아무 일 없음 | **rare+ 확정** + 작심삼일 클럽 "결계 섬데이 돌파" |
| 북 마지막 1개 | 운에 맡김 (무한 가뭄 가능) | 귀해지되 세계관 내 5드랍 내 확정 + "마지막 파편!" 강조 |
| 일련번호 | #1, #2, #3… (서열 노출) | #48291 같은 무작위 고유 번호 |

## 4. 비범위 (Out of Scope)

- 액티비티배지 엔진 변경 (v3 그대로 유지 — 이번 범위는 드랍엔진만)
- 외부 날씨 API 연동 (강수 오버라이드)
