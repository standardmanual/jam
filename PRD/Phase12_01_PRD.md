# JAM! Phase 12 PRD — 잼 포인트 시스템 (1a단계: 체계만)

> 작성일: 2026-07-23
> 설계 원천: `PRD/POINT_SYSTEM_OBJECT_MODEL.md`(개념 모델), `PRD/POINT_SYSTEM_INTERACTION_FLOW.md`(화면 흐름) — 이 PRD는 두 문서에서 이미 확정된 결정을 개발 착수 형태로 정리한 것이다.

---

## 1. 배경 및 목표

### 현행 시스템 분석 결과

**이미 갖춰진 것 (재사용):**
- `missions.reward_type`(`'badge' | 'points' | 'item_badge'`) / `missions.reward_points` — 이미 존재(마이그레이션 011). 미션 상세·피드·프로필 화면 모두 "+100P"로 표시 중.
- `recordFeedEvent()`(`src/lib/activity-feed/`) — 배지 발급(`badge-engine/index.ts:526`), 아이템 드랍(`drop-engine/index.ts:395`), 미션 완료(`missions/checker.ts:92`) 세 지점에서 이미 호출되는 공통 훅. 포인트 지급도 이 세 지점에 나란히 붙인다.
- `abusing_policy` / `drop_policy` 싱글톤 테이블 + 어드민 편집 화면 — "서비스 전체 설정값 1행 + 어드민 폼" 패턴의 선례.
- `admin/users/[id]` — 유저별 상세 화면(배지 발급 히스토리 등 이미 어드민 전용 정보 노출 중).

**문제 (이번에 만드는 것):**
- 포인트를 실제로 지급·차감·조회하는 로직이 전혀 없음 — 화면의 "+100P"는 장식.
- 배지 자체에 포인트 보상값이 없음.
- 운영자가 포인트 유통을 확인·조작할 화면이 없음.

### Phase 12(1a) 목표

**포인트를 언제 주고, 어디에 쌓이는지까지만 만든다. 쓰는 곳(포인트 상점)은 이번 범위가 아니다.**

1. 유저별 포인트 잔액 + 불변 원장(적립/차감 내역)
2. 서비스 전체 발행량을 추적하는 장부(어드민 전용)
3. 배지 발급 시 포인트 지급(배지에 포인트값이 붙어 있을 때) + 미션 완료 보상 포인트 지급(기존 필드 연동)
4. 유저 화면: 내 프로필에서만 보이는 잔액 + 포인트 내역
5. 어드민 화면: 포인트 관리 대시보드 + 유저별 지급/회수(사유 목록 포함) + 배지·미션 에디터 확장

## 2. 확정된 결정사항

| 항목 | 결정 |
|------|------|
| 이번 범위 | **1a(체계)만** — 잔액·원장·발행장부·적립연동·어드민. 포인트 상점(1b), 플리마켓/물물교환(2단계)은 이번에 만들지 않음 |
| 소급 지급 | **안 함** — 기능 배포 이전에 이미 발급된 배지·완료된 미션은 포인트 지급 대상 아님 |
| 포인트-배지 소유권 연동 | **연동 안 함** — 배지 발급 시 1회 영구 지급, 이후 그 배지를 잃어도(조합 소각·드랍 등) 지급된 포인트는 회수하지 않음 |
| 발행량 상한 | **없음** — 발행 장부(Treasury)는 마이너스로 내려가도 됨. 실제 예산이 아니라 "지금까지 얼마나 찍어냈는지 세는 계산기" |
| 어드민 지급/회수 진입점 | **`/admin/users/[id]`와 `/admin/points` 양쪽에 배치**, 실행 로직은 공유 |
| 지급/회수 사유 | **미리 정한 목록 + "기타"(선택 시 자유 입력란 노출)** — 목록: CS 보상 / 오류 정정 / 이벤트·프로모션 지급 / 어뷰징 적발 회수 / 과거 데이터 소급 반영 / 기타 |
| 포인트 잔액 노출 위치 | **내 프로필 화면, 이메일 바로 아래.** 타인이 보는 프로필(`/[username]`)에는 노출 안 함 — 이메일과 같은 급의 비공개 정보 |
| 미션 보상이 배지이고 그 배지가 포인트를 포함하는 경우 | 이중 지급처럼 보일 수 있어 **미션 에디터에 경고 문구** 노출(버그 아님, 의도한 설계임을 운영자가 그 자리에서 확인) |

## 3. 사용자 경험 변화

| 시점 | 지금 (Phase 12 이전) | Phase 12 이후 |
|------|----------------------|----------------|
| 배지 발급 (포인트값 있는 배지) | 배지만 받음 | 배지 + "+N P" 함께 지급, 피드 카드에 표시 |
| 미션 완료 | 화면에 "+100P" 표시되지만 실제로 안 쌓임 | 실제로 잔액에 쌓임 |
| 내 프로필 | 잔액 표시 없음 | 이메일 아래 현재 잔액 상시 노출 |
| 타인 프로필 조회 | (해당 없음) | 상대방 잔액은 보이지 않음 |
| 운영자 CS 대응 | 포인트 관련 조작 수단 없음 | 유저 상세에서 바로 지급/회수 가능, 사유 기록 남음 |
| 운영자 전체 현황 파악 | 불가능 | `/admin/points`에서 총 발행량·유통량·정합성 확인 |

## 4. 비범위 (Out of Scope)

- 포인트 상점(`PointSink`) — 가방 칸 확장 등 포인트 소비 기능 전체 (1b단계)
- 플리마켓(정가 매물) / 물물교환 제안 (2단계)
- 실물 패치 스토어 연동
- 기존 배지·미션 스키마의 파괴적 변경
- 포인트 소급 지급

## 5. 완료 기준

- [ ] `badges.point_reward > 0`인 배지가 발급되면 해당 유저 포인트 잔액이 정확히 증가하고 원장에 기록됨
- [ ] `missions.reward_type = 'points'`인 미션 완료 시 실제로 잔액이 증가함 (기존 화면 표시와 금액 일치)
- [ ] 내 프로필에서만 잔액이 보이고, 타인 프로필에서는 보이지 않음
- [ ] 포인트 내역 화면에서 최근 적립/사용 내역과 각 사유가 정확히 표시됨
- [ ] `/admin/points`에서 총 발행량 · 총 회수량 · 유통량 · 유저 보유 합계가 정확히 계산됨, 어긋나면 경고 배너 노출
- [ ] `/admin/users/[id]`와 `/admin/points` 양쪽에서 동일한 유저 지급/회수를 실행할 수 있음
- [ ] 지급/회수 시 사유 목록에서 선택, "기타" 선택 시에만 자유 입력란 노출, 선택된 사유가 원장에 남음
- [ ] 미션 에디터에서 배지 보상을 선택했을 때 그 배지가 포인트를 포함하면 경고 문구가 노출됨
- [ ] `npx tsc --noEmit` 0 에러
