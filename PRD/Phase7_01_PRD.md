## 1. 기능 개요

### 한 줄 요약
유저가 현재 위치 반경 50m 내 POI를 골라 인벤토리 아이템을 직접 드랍하고, 다른 유저가 드랍한 아이템을 픽업할 수 있는 실시간 위치 기반 교환 시스템.

### 기존 시스템과 차이

| 구분 | 기존 Phase 6 | Phase 7 (신규) |
|------|-------------|----------------|
| 드랍 주체 | 어드민 (관리 패널) | **유저 직접** |
| 픽업 방식 | Strava 경로 자동 교차 검증 | **유저가 직접 선택** |
| 드랍 위치 | 임의 GPS 좌표 | **기존 POI 테이블 기반** |
| 드랍 대상 | 시스템이 생성한 이벤트 | **유저 인벤토리 아이템** |

### 해결하는 문제
- Strava 동기화 없이도 앱을 쓸 이유 (인터랙티브 요소 부재)
- 유저 간 아이템 교류 수단 없음 (인벤토리가 고립됨)
- POI 방문의 게임적 의미 없음

## 2. 사용자 플로우

### 2-1. 드랍 플로우

```
유저: 드랍 버튼 탭
  └─ 위치 권한 요청 (없으면 요청)
      └─ 현재 GPS 좌표 획득
          └─ 반경 50m 내 POI 검색 (DB 쿼리)
              ├─ POI 없음 → "근처에 드랍 가능한 POI가 없어요" 토스트
              └─ POI 있음 → 인앱 구글맵 + POI 마커 표시
                  └─ 유저: POI 선택
                      └─ 인벤토리 아이템 목록 표시
                          └─ 유저: 아이템 선택 + 확인
                              └─ DB: poi_drops INSERT + inventory_items 소프트 삭제
                                  └─ 완료 토스트 "드랍 완료!"
```

### 2-2. 픽업 플로우

```
유저: 픽업 버튼 탭 (또는 드랍 화면에서 전환)
  └─ 현재 GPS 좌표 획득
      └─ 반경 50m 내 아이템 있는 POI 검색
          ├─ 없음 → "근처에 픽업 가능한 아이템이 없어요"
          └─ 있음 → 인앱 구글맵 + POI 마커 표시
              └─ 유저: POI 선택
                  └─ 해당 POI의 드랍된 아이템 목록 표시
                      └─ 유저: 아이템 선택
                          └─ 인벤토리 슬롯 확인
                              ├─ 슬롯 없음 → "인벤토리가 꽉 찼어요"
                              └─ 슬롯 있음 → DB: poi_drops UPDATE + inventory_items INSERT
                                  └─ 완료 토스트 "픽업 완료!"
```

## 3. 기능 상세

### 3-1. 드랍

- **트리거**: 유저가 JAM! 앱에서 드랍 버튼 탭
- **위치 기준**: 브라우저 Geolocation API (getCurrentPosition)
- **POI 검색**: 현재 위치 반경 50m 내 `poi` 테이블 조회
- **지도**: 인앱 Google Maps JavaScript API (마커로 POI 표시)
- **드랍 대상**: 유저 인벤토리(`inventory_items`) 내 아이템 중 1개 선택
- **드랍 제한**: 없음 (인벤토리에 아이템 있으면 언제든 드랍 가능)
- **드랍 후**: 해당 `inventory_items` 레코드 소프트 삭제 (dropped_at 기록), `poi_drops` 에 신규 레코드 생성
- **자기 아이템 본인 픽업**: 불가 (dropper_user_id = 현재 유저 필터링)

### 3-2. 픽업

- **조건**: 유저가 해당 POI 반경 50m 이내에 실제로 있어야 함
- **표시**: POI에 드랍된 아이템 목록 (badge 이름, rarity, 드랍한 유저 display_name, 드랍 시각)
- **선택**: 유저가 목록에서 1개 선택
- **픽업 후**: `poi_drops.picked_up_by` 업데이트, 피커 인벤토리에 아이템 추가
- **인벤토리 슬롯**: 슬롯 부족 시 픽업 차단

### 3-3. 지도 UI

- **라이브러리**: Google Maps JavaScript API (`@googlemaps/js-api-loader`)
- **무료 한도**: 월 $200 크레딧 (약 28,000 지도 로드/월)
- **표시 요소**:
  - 현재 위치 마커 (파란 점)
  - POI 마커 (아이템 있음: 주황/노란색, 없음: 회색)
  - 50m 반경 원 (선택 사항)
- **인터랙션**: POI 마커 탭 → 해당 POI 정보 패널 슬라이드업

## 4. 엣지 케이스

| 상황 | 처리 |
|------|------|
| 위치 권한 거부 | "위치 권한이 필요해요" 안내 + 설정 유도 |
| GPS 정확도 낮음 | accuracy > 100m이면 경고 후 계속 진행 |
| 동시에 두 유저가 같은 아이템 픽업 시도 | DB UNIQUE 제약 또는 낙관적 잠금으로 선착순 처리 |
| 드랍 후 POI 밖으로 이동 | 드랍은 완료 상태 유지 (위치 재확인 없음) |
| 픽업 중 아이템 소진 | "이미 픽업된 아이템이에요" 토스트 |
| 인벤토리 꽉 찬 상태에서 픽업 | 픽업 전 슬롯 사전 확인 후 차단 |

---

## 5. 비기능 요구사항

- **지도 로드 시간**: 3초 이내
- **POI 쿼리 응답**: 500ms 이내 (PostGIS 또는 Haversine 쿼리)
- **API 키 보안**: Maps API 키는 환경변수 + HTTP Referer 제한
- **인증**: Supabase 세션 인증 (미인증 유저 드랍/픽업 불가)

---

## 6. 확정 사항

- **Google Maps API 키**: 신규 발급 필요 (`NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` 환경변수)
- **드랍 만료 정책**: 영구 유지 (`expires_at = NULL`, 회수 없음)
- **드랍 이력**: 유저에게 미노출 / 어드민 패널 유저 정보에 기록
- **POI 마커 디자인**: 기존 디자인 시스템 활용 예정
