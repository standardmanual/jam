# JAM! Phase 8 PRD — 세계관 기반 아이템 배지 시스템

> 작성일: 2026-07-14  
> 범위: 세계관(Faction) 도입, 아이템북 슬로팅 시스템, 드랍 엔진 업그레이드, 어드민/유저 화면 전면 개편

---

## 1. 배경 및 목표

현재 아이템 배지는 100개가 rarity 기반으로 랜덤 드랍되며, 아이템북은 단순한 배지 묶음 레시피로만 존재한다. 세계관(스토리) 개념이 없어 배지 수집의 서사적 맥락이 없다.

Phase 8의 목표:
- **10개 세계관** 도입으로 배지 수집에 스토리와 소속감 부여
- **아이템북 슬로팅** 시스템으로 유저가 능동적으로 컬렉션을 완성하는 경험 제공
- **활동 조건 기반 드랍**으로 "어떤 활동을 해야 어떤 배지가 나온다"는 맥락 부여
- **어드민 완전 제어** — 세계관/아이템북/배지의 모든 속성을 어드민에서 관리

### 2-2. 세계관 어드민 제어 속성

| 속성 | 타입 | 설명 |
|------|------|------|
| `name` | text | 세계관 이름 |
| `tagline` | text | 대표 슬로건 |
| `description` | text | 세계관 서사 (유저 화면 노출) |
| `image_url` | text | 대표 이미지 |
| `drop_weight` | float (0.1~10.0) | 드랍 시 이 세계관 배지 선택 가중치. 기본 1.0. 이벤트 기간 부스트 용도 |
| `is_active` | bool | 비활성화 시 드랍 풀에서 제외 |
| `sort_order` | int | 어드민/유저 화면 노출 순서 |
| `drop_condition_json` | JSONB | 이 세계관의 기본 드랍 활동 조건 (하위 상속) |

`drop_condition_json` 구조:
```json
{
  "activity_types": ["running", "cycling"],
  "min_distance_km": 5,
  "min_elevation_m": 0,
  "min_duration_min": 0
}
```
- `activity_types`: 허용 활동 종류 (빈 배열 = 전체 허용)
- `min_distance_km`: 최소 거리 조건 (0 = 조건 없음)
- `min_elevation_m`: 최소 고도 상승 조건 (0 = 조건 없음)
- `min_duration_min`: 최소 활동 시간 조건 (0 = 조건 없음)

## 3. 아이템 배지 변경

### 3-1. 추가 속성

| 속성 | 타입 | 설명 |
|------|------|------|
| `faction_id` | UUID FK (nullable) | 소속 세계관. NULL = Public 세계관 |
| `item_book_id` | UUID FK (nullable) | 소속 아이템북. NULL = 아이템북 미배정 |
| `drop_weight` | float (0.1~10.0) | 드랍 시 이 배지의 개별 가중치 (같은 rarity 내). 기본 1.0 |
| `drop_condition_json` | JSONB | 이 배지의 개별 드랍 조건. 세계관/아이템북 조건을 이 범위 내에서 좁힘 |

### 3-2. 활동 조건 3단계 상속 규칙

```
세계관 drop_condition_json (최상위 — 허용 범위 정의)
    └── 아이템북 drop_condition_json (중위 — 세계관 범위 내에서 좁힘)
            └── 아이템배지 drop_condition_json (최하위 — 아이템북 범위 내에서 더 좁힘)
```

**충돌 해결**: 하위 설정이 상위보다 더 좁은 조건이어야 함. 더 넓은 조건을 하위에 설정하면 어드민에서 경고 표시.

### 3-3. 배타적 아이템북 소속

- 아이템 배지는 **최대 1개의 아이템북**에만 속할 수 있음
- 아이템북 미배정 배지도 존재 가능 (인벤토리에서 단순 소장용)
- 기존 100개 배지: Public 세계관 배정, 아이템북 미배정 상태로 초기화

## 4. 아이템북 시스템 전면 개편

### 4-1. 아이템북 구조

**최초 설정값**: 10 세계관 × 10 아이템북 × 최대 9개 배지 슬롯 = 최대 900개 배지

어드민이 이후 추가/수정 가능.

### 4-2. 어드민 제어 속성

| 속성 | 타입 | 설명 |
|------|------|------|
| `faction_id` | UUID FK (required) | 소속 세계관 |
| `name` | text | 아이템북 이름 |
| `description` | text | 아이템북 설명 |
| `story_text` | text | 확장 서사 텍스트 (유저 화면 노출) |
| `image_url` | text | 아이템북 대표 이미지 |
| `required_activity_badge_id` | UUID FK (nullable) | 완성에 필요한 액티비티 배지 (선택 사항) |
| `drop_condition_json` | JSONB | 아이템북 드랍 조건 (세계관 조건 상속 후 좁힘) |
| `is_active` | bool | 비활성화 시 유저에게 미노출 |

> **배지 슬롯 수량**: 아이템북에 배정된 아이템 배지 수 = 슬롯 수. 배지를 추가/제거하면 슬롯 수가 바뀜. 별도 `required_item_count` 필드 없음.

### 4-3. 슬로팅 메커니즘 (유저 행동)

1. 유저가 드랍으로 아이템 배지 획득 → 인벤토리에 추가
2. 해당 배지가 아이템북에 배정되어 있으면 → 그 아이템북이 유저 화면에 최초 공개
3. 유저가 아이템북 상세에서 해당 슬롯에 배지를 "추가"
4. 배지는 인벤토리에서 제거되고 아이템북 슬롯으로 이동
5. 모든 슬롯 채워짐 → **완성 트리거** 발화
6. 유저가 슬롯에서 배지를 "제거" → 배지가 인벤토리로 복귀 (완성 상태 취소)

### 4-4. 완성 트리거

- 최초 완성 시 한 번만 발화 (`user_item_book_completions` 기록)
- 이후 완성 취소 후 재완성 → 트리거 재발화 없음 (최초 1회)
- 완성 트리거에 연결 가능한 것: 보상 배지 지급, 미션 완료, 피드 이벤트

## 5. 드랍 엔진 변경

### 5-1. 현재 → 변경 후

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 드랍 풀 | 전체 아이템 배지 (rarity 필터만) | 활동 조건에 맞는 배지만 (세계관/아이템북/배지 조건 평가 후) |
| 가중치 | 없음 (균등 랜덤) | `faction.drop_weight` × `badge.drop_weight` 적용 |
| 드랍 순서 | rarity 추첨 → 랜덤 배지 | rarity 추첨 → 활동 조건 필터 → 가중치 랜덤 배지 |

### 5-2. 드랍 풀 결정 로직

```
활동 완료 (type, distance, elevation, duration 확정)
    ↓
전체 active 아이템 배지 조회
    ↓
세계관 조건 평가 → 통과한 배지만
    ↓  
아이템북 조건 평가 → 추가 필터
    ↓
배지 개별 조건 평가 → 최종 드랍 풀
    ↓
rarity 추첨 (기존 확률표 유지)
    ↓
해당 rarity 배지 중 가중치 랜덤 선택 1개
```

## 6. 유저 화면 변경

### 6-1. 아이템북 목록

- **공개 조건**: 해당 아이템북 소속 배지를 1개 이상 드랍받은 경우에만 노출
- 미공개 아이템북은 목록에서 완전히 숨김 (잠금 아이콘도 없음)
- 새 배지 드랍 시 → 그 배지 소속 아이템북이 신규 공개되면 알림

### 6-2. 아이템북 상세

- 세계관 정보 (이름, 슬로건, 서사)
- 각 슬롯별 필요 배지 표시
  - 보유 중: 배지 이미지 + 인벤토리 수량 표시 + "추가" 버튼
  - 슬롯에 장착됨: 배지 이미지 + "제거" 버튼
  - 미보유: 배지 이미지 흐림 처리 + 획득 힌트 (조건 일부 공개)
- 완성 시 완성 배너 + 보상 표시

### 6-3. 인벤토리 변경

- 아이템북에 장착된 배지: 인벤토리 목록에서 "아이템북 사용 중" 표시 (제거됨)
- 아이템북 미배정 배지: 기존과 동일

## 7. 어드민 화면 추가/변경

### 7-1. 신규: 세계관 관리

- `/admin/factions` — 세계관 목록 (정렬, 활성화 토글)
- `/admin/factions/new` — 세계관 생성
- `/admin/factions/[id]` — 세계관 편집 (drop_weight 슬라이더, 드랍 조건 설정)

### 7-2. 변경: 아이템 배지 관리

- 세계관 선택 드롭다운 추가
- 소속 아이템북 표시 (읽기 전용, 배지에서 직접 변경 불가 — 아이템북 편집에서 관리)
- drop_weight 입력 추가
- 드랍 조건 설정 UI (activity_types 체크박스, 거리/고도/시간 min값 입력)

### 7-3. 변경: 아이템북 관리

- 세계관 선택 드롭다운 추가
- story_text 에디터 추가
- 배지 슬롯 관리: 아이템북에 배정된 배지 목록 + 추가/제거 UI
  - 배지 추가 시 해당 배지의 `item_book_id` 업데이트
  - 배지 제거 시 `item_book_id` NULL로 초기화
- 드랍 조건 설정 UI (세계관 조건 자동 상속 표시 + 개별 override)
- is_active 토글

## 8. 데이터 초기화 계획

Phase 8 런칭 시 일괄 처리:

1. `factions` 테이블에 Public + 10개 세계관 데이터 INSERT
2. 기존 100개 아이템 배지 → `faction_id = public_faction_id` 업데이트
3. 10 × 10 × 9 = 최대 900개 신규 아이템 배지 + 100개 아이템북 일괄 INSERT (CSV 제공 예정)
4. 신규 배지에 각 아이템북 배정 (`item_book_id` 세팅)

> 기존 유저 인벤토리의 배지는 영향 없음. 기존 100개 배지는 Public 세계관으로 이동하고 아이템북 미배정 상태 유지.
