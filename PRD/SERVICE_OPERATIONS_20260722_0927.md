## [버그 수정] 인벤토리·조합 재료 목록의 슬롯 장착 아이템 필터링

### 원인

`user_item_book_slots` API(`POST`/`DELETE` — `src/app/api/itembooks/[id]/slot/route.ts`)는 슬롯 장착/해제 시 `inventory_items.slotted_in`을 이미 정확히 갱신하고 있었다. 문제는 **화면 조회 쿼리들이 이 필드를 걸러내지 않아**, 슬롯에 넣은 아이템이 인벤토리 목록에도 계속 표시되던 것 — DB 상태는 정상, 조회 로직만 결함.

### 수정

| 파일 | 변경 |
|------|------|
| `src/app/(main)/inventory/page.tsx` | 인벤토리 아이템 필터에 `slotted_in === null` 조건 추가 |
| `src/app/(main)/combine/page.tsx` | 조합 재료 조회 쿼리에 `.is('slotted_in', null)` 추가 — 슬롯 장착 아이템은 조합 재료로 사용 불가 |

### 규칙 확정

- 아이템배지는 인벤토리 또는 아이템북 슬롯 중 **한 곳에만** 위치 (동시 표시 없음)
- 슬롯 장착 → 인벤토리·조합 재료 목록에서 제거
- 슬롯 해제 → 인벤토리 목록에 자동 복귀 (별도 로직 불필요, 필터 조건만으로 처리됨)

### 운영 유의사항

- `src/app/(main)/badges/page.tsx`(배지 컬렉션 탭)는 의도적으로 그대로 둠 — 슬롯 이동 여부와 무관하게 "보유한 배지"를 보여주는 화면이라 이번 수정 범위 아님. 슬롯에 넣은 배지도 컬렉션에는 계속 표시되는 것이 맞음.
- 스키마 변경 없음 — 배포만으로 즉시 적용, 재초기화 불필요.
