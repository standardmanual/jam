## 0. 현황 진단

### 현행 로직
활동 1건 → rarity 추첨(80% 드랍 / 20% 꽝) → **전체 활성 아이템북의 해당 rarity 배지 중** drop_weight 가중 랜덤 → 인벤토리 삽입.

### 게이미피케이션 관점 문제점

| 문제 | 원인 | 심리적 결과 |
|------|------|------------|
| **보상 인플레이션** | 활동당 80% 드랍 — 사실상 고정비율(fixed ratio)에 가까움 | 보상 감가(reward decay). "또 나왔네" → 드랍의 특별함 소멸 |
| **서사 단절** | 900개 풀에서 완전 랜덤 | 직전에 '낭만 미식가' 배지를 주웠는데 다음에 '장비병 환자' 파편 → 세계관 몰입 형성 불가 |
| **분산·산만** | 유저가 동시에 10개 세계관 100개 아이템북을 0~9% 진행 | 진행 심리학의 핵심인 "완성이 보이는 거리감" 상실 → 흥미 소실 |
| **완성 페이싱 부재** | 같은 아이템북 배지가 연속으로 나올 수도, 영원히 안 나올 수도 | 너무 빠른 완성 = 허무 / 마지막 1개 무한 가뭄 = 좌절. 양쪽 모두 통제 불가 |
| **드랍의 무의미성** | 활동 맥락(날씨·시간·종목)과 드랍 내용 무관 | 폭우 속 러닝 후 '먼지 쌓인 덤벨'이 나오면 보상이 정보가(informational value)를 갖지 못함 |

## 1. 설계 원칙 (Engagement Loop 기준)

```
TRIGGER(운동 완료·싱크) → ACTION(싱크 확인) → REWARD(드랍) → INVESTMENT(아이템북 진행·수집)
```

1. **변동비율 + 바닥 보장**: 드랍은 예측 불가해야 설레지만(variable ratio), 가뭄은 유한해야 한다(pity).
2. **보상은 정보여야 한다**: "내가 한 활동"과 "떨어진 파편"이 맥락으로 연결될 때 외적 보상이 내적 동기를 침식하지 않는다 (overjustification 방어).
3. **좁게 집중, 넓게 탐험**: 동시에 추적하는 세계관은 2~3개로 제한. 나머지는 '아직 만나지 못한 세력'으로 남겨 장기 콘텐츠화.
4. **완성은 축하하되, 너무 쉬우면 안 된다**: 아이템북 완성 직전 구간은 의도적으로 페이싱을 늦추되, 상한을 둬 좌절로 넘어가지 않게 한다.
5. **다크패턴 금지**: 미접속 페널티 없음, 드랍 확률 구조는 감춤 없이 일관, 일일 상한으로 어뷰징·과몰입 동시 방어.

## 2. 3-레이어 드랍 로직

드랍 결정을 3단계로 분리한다: **① 드랍 발생 여부 → ② 세계관 선택 → ③ 아이템북·배지 선택**

### Layer 1 — 드랍 발생 (빈도 최적화)

| 파라미터 | 값(초기) | 근거 |
|----------|---------|------|
| 기본 드랍률 | 활동당 **40%** | 현행 80%의 절반. "안 나올 수도 있다"가 기대감을 만든다 |
| Pity(바닥 보장) | **3회 연속 꽝이면 4번째 활동 드랍 확정** | 변동비율의 단점(무한 가뭄) 차단. 기대 드랍 간격 ≈ 1.8활동 |
| 일일 드랍 상한 | **2개/일** | 다회 싱크·짧은 활동 반복(metric gaming) 방어 + 과잉 보상 방지 |
| 주간 첫 활동 보너스 | 그 주 첫 활동은 드랍률 **+20%p** | 주간 루프의 재진입 트리거 |
| 복귀 보너스 | **7일+ 미활동 후 복귀 활동은 드랍 확정** + 활성 세계관 배지 우선 | 복귀 순간이 가장 이탈 위험이 큼. "돌아오니 세계관이 나를 기억한다" 경험 |
| rarity 분포 | 드랍 발생 시 common 55 / rare 30 / legendary 11 / mythic 4 (%) | 드랍 자체가 희소해진 만큼 rarity 꽝 구간 제거 (현행은 80% 중 세분) |

> 첫싱크(온보딩)는 예외: 기존 "10초 첫 보상" 철학대로 첫 싱크 드랍은 확정 유지.

### Layer 2 — 세계관 선택 (서사 모멘텀)

유저별 상태 `active_worlds`(활성 세계관 2~3개)와 `last_drop_world`(직전 드랍 세계관)를 기준으로 가중 추첨:

| 버킷 | 확률 | 의미 |
|------|------|------|
| **모멘텀**: 직전 드랍과 같은 세계관 | **50%** | "이야기가 이어진다"는 감각의 핵심 |
| **인접**: 직전 세계관과 콘텐츠로 연결된 세계관 (§3 인접 그래프) | **25%** | 같은 세계관 반복의 지루함 방지 + 크로스오버 서사 |
| **탐험**: 활성 세계관 밖 랜덤 (신규 세계관 발견) | **15%** | 신선함 주입. 이 드랍이 3회 누적되면 해당 세계관을 활성 후보로 제안 |
| **맥락 오버라이드** (조건 충족 시 최우선) | **10%** | §4 참조 — 활동 맥락과 정합하는 세계관 강제 |

**활성 세계관 규칙:**
- 신규 유저: 첫 3회 드랍은 온보딩 성격의 세계관 2개로 고정 — **작심삼일 클럽 + 유저 주 활동종목 매핑 세계관** (걷기→숲속의 갱단, 러닝→비트 마에스트로, 사이클→장비병 환자들, 등산/트레일→아스팔트 레인저). '작심삼일 클럽'은 초심자 자기서사와 정확히 공명하고, "결계 섬데이 돌파"가 곧 유저 자신의 이야기가 된다.
- 활성 세계관은 최대 3개. 아이템북 1권 완성 시 유저에게 **다음 세계관 선택권** 제공 (autonomy — 시스템이 정해주지 않고 유저가 고름).
- **미스터리 헌터는 활성/모멘텀 규칙 예외**: legendary·mythic 드랍 시에만 낮은 확률로 어느 유저에게나 등장하는 전역 스파이스. "도시 괴담" 포지션 유지.

### Layer 3 — 아이템북·배지 선택 (완성 페이싱)

선택된 세계관 안에서 아이템북을 고를 때, **완성도 기반 감쇠 가중치**를 적용한다:

```
아이템북 가중치 = base_weight × (1 − completion × 0.7)
  completion = 해당 유저의 그 아이템북 수집률 (0.0~1.0)
```

- 0% 진행 북: 가중치 100% → 새 북 진입은 쉽게
- 50% 진행 북: 가중치 65% → 중반은 완만하게
- 89% 진행 북(마지막 1개): 가중치 ~38% → 막바지는 귀하게

**마지막 조각 규칙 (near-completion 심리 활용 + 좌절 방지):**
- 한 아이템북의 마지막 1개 배지는 위 감쇠로 자연 드랍이 어렵다 → 긴장감 형성 (progress bar psychology: 마지막 10%가 가장 길게 느껴져야 완성이 짜릿하다)
- 단, **그 세계관에서 5회 드랍이 더 발생할 때까지 못 얻으면 다음 드랍에서 확정** (마지막 조각 pity). 좌절 상한선.
- 마지막 조각이 떨어질 때는 UI에서 "○○ 아이템북의 마지막 파편!" 강조 → 완성 순간을 공유 가능한 milestone moment로.

**동일 북 연속 방지:** 직전 드랍과 같은 아이템북은 가중치 추가 ×0.5 — 같은 세계관 안에서도 북이 번갈아 차오르게 하여 "북 하나가 순식간에 완성"되는 것을 구조적으로 차단.

## 3. 세계관 인접 그래프 (콘텐츠 연결성)

세계관 문서의 서사에서 직접 도출한 인접 관계. Layer 2의 "인접" 버킷은 이 그래프의 이웃에서 추첨한다.

| 세계관 | 인접 세계관 | 서사 근거 (문서 인용 기반) |
|--------|------------|--------------------------|
| 낭만 미식가 | 숲속의 갱단, 포션 연금술사, 작심삼일 클럽 | 흘린 간식이 "은밀한 포식자들의 표적" = 갱단이 훔쳐감 / 먹는 것·마시는 것 연결 / 회식·야식 정서 |
| 아스팔트 레인저 | 미스터리 헌터, 장비병 환자들, 셔터 마피아 | 극한 환경 ↔ 한계 돌파 / 혹독한 환경에서 장비 전사 / 악천후 풍경 출사 |
| 숲속의 갱단 | 낭만 미식가, 아날로그 수집가, 미스터리 헌터 | 음식 절도 / "까치의 반짝이는 보물" ↔ 반짝이는 잡동사니 수집 / 심야 숲의 기묘함 |
| 셔터 마피아 | 아날로그 수집가, 비트 마에스트로, 아스팔트 레인저 | 필름 카메라·아날로그 감성 / 새벽 감성(물안개 ↔ Lo-Fi) / 극한 풍경 |
| 장비병 환자들 | 작심삼일 클럽, 포션 연금술사, 아스팔트 레인저 | "쇼핑만 프로 선수" = 동전의 양면 / 화학적 최적화 집착 동질성 / 기어의 무덤 |
| 비트 마에스트로 | 셔터 마피아, 미스터리 헌터, 포션 연금술사 | 감성 크리에이터 / "노이즈 캔슬링 0" ↔ "침묵의 메아리" 차원 서사 / 러너스 하이 부스팅 |
| 포션 연금술사 | 낭만 미식가, 장비병 환자들, 비트 마에스트로 | 먹고 마시는 것 / 최적화 광기 / 러너스 하이 EDM ↔ 부스터 |
| 아날로그 수집가 | 미스터리 헌터, 숲속의 갱단, 셔터 마피아 | **에필로그 공식 크로스오버** (멈춘 초시계 ↔ 회중시계) / 잡동사니 수집 / 필름 감성 |
| 작심삼일 클럽 | 낭만 미식가, 장비병 환자들, (졸업: 임의 세계관) | 회식·치킨 서사 / 장비만 사는 자들 / "섬데이 돌파" 시 유저가 다음 세계관 선택 = 서사적 졸업 |
| 미스터리 헌터 | (전역 — 모든 세계관과 인접) | 문서상 "모든 유저들이 열망하는 환상 등급 전리품" = 전역 스파이스 포지션 |

> 인접 그래프는 DB 테이블(`world_adjacency`) 또는 세계관 마스터의 JSON 컬럼으로 관리해 운영자가 시즌별로 조정 가능하게 한다.

## 4. 맥락 오버라이드 (드랍을 '정보'로 만들기)

활동의 실제 맥락이 특정 세계관과 강하게 정합하면, 그 드랍은 세계관 추첨을 건너뛰고 해당 세계관에서 나온다. **이게 드랍 시스템의 킬러 경험이다** — "폭우 속에서 뛰었더니 아스팔트 레인저의 파편이 나왔다"는 보상을 성취의 증거(informational reward)로 만들어 내적 동기를 보호한다.

| 활동 맥락 (이미 수집 중인 데이터) | 오버라이드 세계관 | 예시 아이템북 |
|--------------------------------|-----------------|--------------|
| 강수 중 활동 / 태풍·폭우 | 아스팔트 레인저 | 폭우 속의 질주, 태풍의 눈 |
| 기온 ≤ -10°C / ≥ 33°C | 아스팔트 레인저 | 영하 15도의 호흡, 아스팔트 열기 |
| 새벽 시간대 (05~07시) | 비트 마에스트로 / 셔터 마피아 | 새벽의 Lo-Fi, 새벽 물안개 몽환 |
| 심야 시간대 (23~04시) | 낭만 미식가 / 숲속의 갱단 | 편의점 심야 만찬, 은밀한 밤의 무도회 |
| 고고도 상승 (elevation_gain 상위) | 낭만 미식가 / 비트 마에스트로 | 업힐 끝의 단맛, 업힐의 하드락 |
| 7일+ 공백 후 복귀 활동 | **작심삼일 클럽** | 결계 '섬데이' 돌파 |
| 러너스 하이 조건 (고강도 장시간) | 미스터리 헌터 (rare 이상 한정) | 차원의 틈새 |

- 오버라이드 발동률은 조건 충족 시 60% (항상 발동하면 예측 가능해져 변동성이 죽는다).
- 복귀 활동의 작심삼일 클럽 오버라이드는 **가장 우선순위가 높다** — "오늘 드디어 침대의 중력을 이겨냈다"(문서 원문)를 시스템이 정확히 그 순간에 건네는 것.

## 5. 데이터 모델 변경안

```sql
-- 유저별 드랍 상태 (신규)
CREATE TABLE user_drop_state (
  user_id uuid PRIMARY KEY REFERENCES users(id),
  active_world_ids text[] NOT NULL DEFAULT '{}',   -- 활성 세계관 (최대 3)
  last_drop_world_id text,                          -- 직전 드랍 세계관
  last_drop_book_id uuid,                           -- 직전 드랍 아이템북
  pity_counter int NOT NULL DEFAULT 0,              -- 연속 꽝 카운터
  last_piece_pity jsonb NOT NULL DEFAULT '{}',      -- {book_id: 카운터} 마지막 조각 pity
  daily_drop_count int NOT NULL DEFAULT 0,
  daily_drop_date date,
  exploration_hits jsonb NOT NULL DEFAULT '{}',     -- {world_id: 탐험 드랍 누적} → 3회 시 활성 제안
  updated_at timestamptz NOT NULL DEFAULT now()
);

-- 세계관 마스터 (item_books에 world 컬럼이 없다면 신규, 있다면 인접만 추가)
--   worlds(id, name, adjacency jsonb)  또는  item_books.world_id + world_adjacency 테이블
```

기존 `item_books`에 세계관 소속 컬럼(`world_id`)이 없으면 추가 필요 — 레시피 xlsx의 "세계관" 열이 원천 데이터.

### 엔진 의사코드

```
tryItemDrop(userId, activity):
  state = user_drop_state 조회/초기화
  1. 일일 상한 체크 (2/일) → 초과 시 종료
  2. 드랍 판정: 복귀(7일+)면 확정 / pity_counter ≥ 3이면 확정
     / 주간 첫 활동이면 60% / 기본 40% → 꽝이면 pity_counter++ 후 종료
  3. rarity 추첨 (55/30/11/4) + 섀도우밴 체크(기존 유지)
  4. 세계관 선택:
     a. 맥락 오버라이드 조건 매칭 → 60% 확률로 해당 세계관 확정
     b. 아니면 모멘텀 50 / 인접 25 / 탐험 15 / (오버라이드 미발동분 흡수 10→모멘텀)
  5. 아이템북 선택: 세계관 내 활성 북 대상
     가중치 = drop_weight × (1 − completion × 0.7) × (직전 북이면 0.5)
     단, 마지막 조각 pity 도달 북이 있으면 그 북의 마지막 배지 확정
  6. 배지 선택: 북 내 미보유 배지 우선 (rarity 일치 없으면 인접 rarity 폴백)
     — 마지막 1개 남은 북이면 §2 마지막 조각 규칙 적용
  7. 인벤토리 삽입 + state 갱신 (기존 로직 유지)
```

> 기존 `isDroppableForActivity`(누적조건 제외 가드), 섀도우밴, 인벤토리 슬롯, 유효기간 로직은 그대로 유지 — 이 설계는 "후보 풀 구성과 추첨"만 교체한다.

## 6. 파라미터 요약표 (런칭 후 튜닝 대상)

| 파라미터 | 초기값 | 튜닝 신호 |
|----------|--------|----------|
| 기본 드랍률 | 40% | 주간 활동 유저의 드랍 체감이 "가끔 안 나옴" 수준인지 |
| pity 임계 | 3회 | 연속 꽝 불만 발생 시 ↓ |
| 일일 상한 | 2 | 다회 싱크 유저의 드랍 편중 시 ↓ |
| 모멘텀/인접/탐험 | 50/25/15(+10) | "같은 것만 나온다" 피드백 시 모멘텀 ↓ |
| 완성 감쇠 계수 | 0.7 | 첫 북 완성까지 기대 드랍 수 12~18개 목표 (10개 북 ×9개 배지 기준, 중복·분산 감안) |
| 마지막 조각 pity | 세계관 내 5드랍 | 완성 직전 이탈률 관찰 |
| 오버라이드 발동률 | 60% | 맥락 드랍의 "우연 같은 필연" 체감 유지 |

## 7. 성공 지표

- **핵심**: D7/D30 리텐션 (드랍 v2 코호트 vs v1), 첫 아이템북 완성까지 도달률·소요일
- **서사 체감**: 연속 드랍의 동일/인접 세계관 비율 (목표 70%+), 아이템북 상세 페이지 방문율
- **페이싱 건전성**: 아이템북 완성 소요 드랍 수 분포 (너무 빠른 완성 <8드랍 비율 5% 미만), 마지막 조각 대기 중 이탈률
- **경고 신호** (sharp edges): 짧은 활동 반복 패턴 증가(metric gaming), 드랍 없으면 활동 안 하는 유저 비율(motivation crowding) — 발견 시 맥락 오버라이드 비중 강화로 보상의 정보성 회복

## 8. 윤리 체크리스트

- [x] 미접속 페널티 없음 — 복귀는 오히려 확정 드랍으로 환영
- [x] 드랍을 위한 무한 반복 유도 없음 — 일일 상한이 자연 종료점(completion point) 제공
- [x] 활동의 내적 가치 보존 — 맥락 오버라이드가 보상을 성취의 '증거'로 유지
- [x] 유저 통제권 — 다음 활성 세계관은 유저가 선택
- [x] 투명성 — pity·상한 구조는 도움말에 공개 가능한 수준으로 단순

## 9. 단계별 적용 제안

1. **Phase A (스키마·데이터)**: `item_books.world_id` 부여 + 세계관 마스터·인접 그래프 시드 + `user_drop_state` 테이블
2. **Phase B (빈도)**: Layer 1만 교체 (드랍률 40% + pity + 일일 상한 + 복귀 확정) — 리스크 최소, 효과 즉시 측정 가능
3. **Phase C (서사)**: Layer 2·3 (모멘텀·인접·완성 감쇠·마지막 조각) 적용
4. **Phase D (맥락)**: 맥락 오버라이드 — 날씨·시간 데이터는 배지엔진이 이미 사용 중이므로 드랍엔진에 연결만
