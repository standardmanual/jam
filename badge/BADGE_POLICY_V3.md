## 1. 설계 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **첫 싱크 Common 전용** | 첫 Strava 동기화 시 누적 스탯이 아무리 높아도 Common 등급만 발급. 이후 싱크부터 정상 평가 |

## 3. 크로스-어트리뷰트 선행 배지 매핑

> 아래 배지 이름 중 **하나라도** Common 등급을 보유해야 Rare 이상 발급 가능

| 카테고리 | 선행 배지 (OR 조건) | 설계 의도 |
|----------|---------------------|-----------|
| `정규(걷기)` | `시간의 마술사`, `작심삼일의 파괴자` | 거리를 늘리려면 시간 지속 또는 연속성 입증 먼저 |
| `정규(러닝)` | `리듬의 발견`, `작심삼일의 파괴자` | 거리 확장 전 속도 또는 루틴 습관 증명 |
| `정규(자전거)` | `언덕의 도전자`, `리듬의 발견` | 거리 확장 전 오르막 극복 또는 속도 역량 증명 |
| `정규(고도)` | `첫 숨결`, `두 바퀴의 시작` | 고도 훈련 전 러닝 또는 자전거 입문 먼저 |

## 4. 등급별 발급 조건 요약

| 등급 | 첫 싱크 발급 | 선행 배지 필요 | 비고 |
|------|-------------|----------------|------|
| Common | ✅ 허용 | ❌ 불필요 | 첫 활동 진입점 |
| Rare | ❌ 차단 | ✅ 필요 (다른 속성 Common) | 두 번째 싱크부터 평가 |
| Legendary | ❌ 차단 | ✅ 필요 | 진행 트랙 심화 단계 |
| Mythic | ❌ 차단 | ✅ 필요 | 최상위 달성 |

## 5. 진행 트랙 예시

### 러닝 유저 진행 순서

```
첫 싱크:
  → 첫 숨결 Common (러닝 3km 누적)

이후 싱크:
  → 리듬의 발견 Common (속도 8km/h)  ← 러닝 Rare 해금 선행 조건
  → 첫 숨결 Rare (러닝 4km) ← 이제 발급 가능

더 이후:
  → 작심삼일의 파괴자 Common (연속 3일) ← 러닝 Legendary 해금 강화
  → 첫 숨결 Legendary (러닝 5km)
```

### 자전거 유저 진행 순서

```
첫 싱크:
  → 두 바퀴의 시작 Common (자전거 10km)

이후 싱크:
  → 언덕의 도전자 Common (고도 50m) 또는 리듬의 발견 Common (속도 8km/h)
  → 두 바퀴의 시작 Rare (자전거 15km) ← 선행 조건 충족 후 발급

더 이후:
  → 페달의 노래 Common → Rare → ...
```

## 7. 엔진 연동 참고

배지를 DB에 생성할 때 `condition_json`에 `prerequisite_badge_names` 배열로 설정:

```json
{
  "activity_type": "road_running",
  "distance_km": 4.0,
  "prerequisite_badge_names": ["리듬의 발견", "작심삼일의 파괴자"]
}
```

엔진은 `prerequisite_badge_names` 중 하나라도 보유하면 통과, 아니면 missed 처리.  
자세한 구현은 `SERVICE_OPERATIONS.md` Section 4 참조.
